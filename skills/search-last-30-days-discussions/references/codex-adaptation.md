# Codex 适配说明

这个 skill 是对上游 [`mvanhorn/last30days-skill`](https://github.com/mvanhorn/last30days-skill) 的中文包装壳，不是完整长期分叉。

## 设计目标

- 保留上游能力边界：研究“最近 30 天”跨平台真实讨论
- 改写上游客户端专属命令：不依赖 `/last30days`、Claude Code、OpenClaw
- 适配 Codex 的工作方式：优先联网检索、交叉核对、输出中文结构化简报

## 建议覆盖的来源

- 高优先级：Reddit、X、YouTube、Hacker News、GitHub、网页
- 视可用性补充：Polymarket、Bluesky、TikTok、Instagram、Threads

## 来源能力与降级策略

- Reddit / Hacker News / GitHub / 网页：
  - 通常最容易直接获取
  - 没有额外账号时也经常能拿到有效信息
- X：
  - 可能需要登录态、API、第三方检索能力或公开索引页面
  - 若拿不到有效结果，明确写出 X 覆盖不足，不要伪装成已覆盖
- YouTube：
  - 视频搜索常可做，但完整转录、评论或批量抓取可能受限
  - 至少提取标题、发布时间、频道、描述和公开可见的重点
- Polymarket：
  - 适合做市场赔率或预期信号
  - 若无相关市场，直接说明无有效市场信号
- TikTok / Instagram / Threads / Bluesky：
  - 常受 API、登录态或第三方工具限制
  - 有能力时作为补充信号；没有时不阻塞主任务

## Codex 执行约束

- 用户提到“最近 30 天”“最近一个月”“最新”“现在”，必须联网核实。
- 技术主题优先引用官方仓库、release、issue、discussion、官方博客和官方文档。
- 争议性结论必须尽量给出对应讨论出处，而不是只给抽象判断。
- 对比查询应保持维度一致：谁更热、谁被夸、谁被吐槽、谁在加速发布、谁的社区更买账。

```md
- ...

信号强弱
- 强信号：...
- 弱信号：...
```

## 不要做的事

- 不要把它退化成普通网页搜索总结
- 不要假装已经覆盖所有平台
- 不要把社区观点写成已证实事实
- 不要忽略时间窗，尤其是“最近 30 天”这一核心约束
